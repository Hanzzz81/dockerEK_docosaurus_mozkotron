Hierarchie obsahu v docs/

STRUKTURÁLNÍ SLOŽKA
📁 01-finance/
Prefix: číslo (01-, 02-...)
Účel: organizace a řazení
Nemá vlastní obsah
Agreguje data z dětí
_category_.json: label, position
KAPITOLA
📖 Saldokonto/
Prefix: písmeno (velké/malé)
Účel: jeden trackovaný celek
Má index.md s popisem
Části: 01-xxx.md, 02-yyy.md + HTML
_category_.json: unit, layer, category, audience
POD-KAPITOLA
📖 Upominky/
Prefix: písmeno (stejné pravidlo)
Účel: složitější část kapitoly
Vnořená uvnitř rodičovské kapitoly
Vlastní index.md + části
Trackuje se nezávisle

Tři typy dokumentů (layer)

TypNázevÚčelPovinné části
SOP Standard Operating Procedure Člověk má něco udělat krok za krokem Hlavička → Mapa procesu (5+ kroků) → Kroky → Varování → Quiz → Potvrzení
BIMG Big Image (dokumentace) Člověk má pochopit, jak něco funguje Hlavička → Vizuální mapa (vždy) → Obsahové sekce → Kontext inline
CHL Cheat List (reference) Rychle najít konkrétní údaj Hlavička → Tabulka/seznam → Hledání
